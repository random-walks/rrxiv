# rrvix Improvement Proposals (RRPs)

Modeled on Python PEPs and Bitcoin BIPs. Substantive changes to the rrvix
protocol — schema modifications, new annotation types, governance changes,
licensing changes — go through the RRP process.

## Lifecycle

```
draft → discussion → (accepted | rejected | withdrawn)
```

Accepted RRPs are immutable. Modifications require a superseding RRP that
references the original.

## Process

1. Copy `0000-template.md` to `NNNN-short-name.md` where `NNNN` is the next
   sequential number.
2. Fill in the template.
3. Open a pull request adding the file.
4. Discussion happens in the PR.
5. Maintainers mark the RRP as accepted, rejected, or withdrawn.

## Numbering

RRPs are numbered sequentially. Numbers are assigned at PR-merge time, not at
draft time, to avoid number collisions.
